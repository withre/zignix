#+title: zignix
#+html: <p><img src="https://img.shields.io/badge/Status-experimental-orange" alt="experimental"> <img src="https://img.shields.io/badge/Zig-any%20build-orange?logo=zig" alt="Zig any build"></p>

A Nix overlay for Zig that lets you pin *any* upstream build — including
nightlies that haven't been ingested into curated source lists yet.

* Why

The popular [[https://github.com/mitchellh/zig-overlay][mitchellh/zig-overlay]] is a curated mirror: it serves whatever
its update bot has imported into ~sources.json~. When you want a Zig
nightly that just landed upstream, you wait for the bot to catch up
(sometimes days). For projects tracking the unreleased ~master~ line
closely, that's a real source of friction.

*zignix* takes a different approach: the consumer flake supplies
~{ version, sha256 }~ directly, and zignix synthesises the upstream URL
and produces a derivation. No PR to upstream required — just paste in
the shasum from [[https://ziglang.org/download/index.json][ziglang.org/download/index.json]].

* Quickstart

** Add zignix to your flake

The lib index is keyed by system, so use ~pkgs.stdenv.hostPlatform.system~
from the appropriate ~pkgs~ for each output rather than threading a bare
~system~ string through your flake. This keeps the same expression valid
under cross-compilation and across all supported hosts.

#+begin_src nix
{
  inputs.zignix.url = "github:youruser/zignix";
  # ... your other inputs ...

  outputs = { self, nixpkgs, zignix, ... }:
    let
      systems = [ "x86_64-linux" "aarch64-linux" "x86_64-darwin" "aarch64-darwin" ];
      forAllSystems = nixpkgs.lib.genAttrs systems;

      zigFor = pkgs:
        zignix.lib.${pkgs.stdenv.hostPlatform.system}.fromBuild {
          version = "0.17.0-dev.607+456b2ec07";
          sha256  = "19275107de7b89ec33d29b50f00997c1381c524d1e33b728472dcbd551da2e33";
        };
    in {
      devShells = forAllSystems (system:
        let pkgs = nixpkgs.legacyPackages.${system}; in {
          default = pkgs.mkShell { packages = [ (zigFor pkgs) ]; };
        });
    };
}
#+end_src

** Bootstrap a new version

Don't know the sha256 yet? Set it to ~lib.fakeHash~ (or any obviously
wrong value) and let Nix tell you the right one:

#+begin_src nix
(zignix.lib.${pkgs.stdenv.hostPlatform.system}.fromBuild {
  version = "0.17.0-dev.612+abcdef012";
  sha256  = nixpkgs.lib.fakeHash;
})
#+end_src

Run ~nix build~ — Nix will fail with the actual hash, which you paste
back in.

Or look it up directly:

#+begin_src bash
curl -s https://ziglang.org/download/index.json \
  | jq -r '.master | .version, ."x86_64-linux".shasum'
#+end_src

* API

All helpers live under ~zignix.lib.${system}~.

** ~fromBuild { version, sha256, urlBase? }~

The headline use case. Synthesises the URL
~https://ziglang.org/builds/zig-<arch>-<os>-<version>.tar.xz~ and
produces a derivation.

This is the right entry point for *nightly* (~master~) builds — those
all live under ~/builds/~.

** ~fromTagged { version, sha256 }~

Sugar for tagged releases, which live at
~https://ziglang.org/download/<version>/~ rather than ~/builds/~.

#+begin_src nix
(zignix.lib.${pkgs.stdenv.hostPlatform.system}.fromTagged {
  version = "0.16.0";
  sha256  = "70e49664a74374b48b51e6f3fdfbf437f6395d42509050588bd49abe52ba3d00";
})
#+end_src

** ~fromUrl { version, url, sha256 }~

Lowest-level escape hatch — for mirrors, self-hosted builds, or any URL
that doesn't fit the standard ziglang.org layout.

#+begin_src nix
(zignix.lib.${pkgs.stdenv.hostPlatform.system}.fromUrl {
  version = "0.17.0-dev.612-mybuild";
  url     = "https://my-mirror/zig-x86_64-linux-0.17.0-dev.612+abc.tar.xz";
  sha256  = "...";
})
#+end_src

* Comparison

| Feature                              | mitchellh/zig-overlay | zignix             |
|--------------------------------------+-----------------------+--------------------|
| Tagged releases (0.x.y)              | ✅                    | ✅ (~fromTagged~)  |
| Nightly already in upstream index    | ✅                    | ✅ (~fromBuild~)   |
| Nightly *not yet* in upstream index  | ❌                    | ✅ (~fromBuild~)   |
| Self-hosted / mirror builds          | ❌                    | ✅ (~fromUrl~)     |
| Static sources.json (offline-OK)     | ✅                    | ✅ (per-project)   |
| Cached aggregated `master-<date>`    | ✅                    | ❌ (out of scope)  |

zignix is intentionally smaller in scope. If you want a one-stop
catalogue of every Zig release ever cut, use mitchellh's. If you want
the smallest surface area that lets you pin *any* upstream build the
day it lands, use zignix.

* Supported systems

- ~x86_64-linux~
- ~aarch64-linux~
- ~x86_64-darwin~
- ~aarch64-darwin~

Other systems are not currently mapped; PRs welcome — see ~nix/lib/default.nix~,
the ~systemMap~ attrset.
