** 🌄 Overview

*zignix* is a tiny Nix helper for pinning Zig binaries by ~version~ and
Nix ~sha256~.

It exists for the moment when a Zig nightly is already available from
upstream, but a curated overlay has not indexed it yet. Your project owns
the exact Zig pin; zignix only turns that pin into the expected Zig binary
package.

** 🚀 Quick Start

This example uses the latest known upstream ~master~ build for
~x86_64-linux~.

#+begin_src nix
  # nix/packages/zig/default.nix
  { pkgs, inputs, ... }:
  
  inputs.zignix.lib.${pkgs.stdenv.hostPlatform.system}.fromBuild {
    version = "0.17.0-dev.813+2153f8143";
    sha256 = "sha256-sNRv/EWHuejdC1JO5bxNoeZ/KLulXnxTTOxkry8tenQ=";
  }
#+end_src

Use the hash for the system you build on. See *Installation* for flake
patterns and *Pin values* for hash lookup commands.

** 📍 Current pins

The repo ships two prebuilt packages, kept current automatically:

| Package      | Tracks              | Version                  |
|--------------+---------------------+--------------------------|
| ~zig-master~ | latest ~master~       | ~0.17.0-dev.813+2153f8143~ |
| ~zig-0_16~     | latest 0.16 release | ~0.16.0~                   |

These are bumped automatically every 6 hours by the repo's update
workflow. Consumers pinning their own version with ~fromBuild~ or
~fromTagged~ are unaffected.

** ⚖️ Comparison

[[https://github.com/mitchellh/zig-overlay][mitchellh/zig-overlay]] is the right choice when you want a curated
catalogue of Zig builds ready to reference.

zignix is intentionally narrower: it gives you a small function that
builds the upstream URL from your explicit ~version~ and ~sha256~. That
means a new nightly can be used as soon as Zig publishes it, without
waiting for this repository or any overlay index to update.

| Need                                | zig-overlay | zignix           |
|-------------------------------------+-------------+------------------|
| Full curated catalogue of releases  | ✅          | ❌               |
| Ready-to-use latest packages        | ✅          | ✅ (master/0.16) |
| Pin a just-published upstream build | Depends     | ✅               |
| Project-owned version/hash          | ❌          | ✅               |
| Custom mirror URL                   | ❌          | ✅               |
| Install versions side by side       | ✅          | ✅ (~withName~)    |

** 🧪 Try it

#+html:<details>
#+html:<summary><b>Run with <code>nix shell</code></b></summary>
#+html:<br>

Run the bundled latest Zig package directly from GitHub:

#+begin_src sh
  nix shell github:withre/zignix#zig-master -c zig version
#+end_src

From a local checkout, use the local flake instead:

#+begin_src sh
  nix shell .#zig-master -c zig version
#+end_src

The printed version should match the pin in the Quick Start example.

#+html:</details>

#+html:<details>
#+html:<summary><b>Build the example project</b></summary>
#+html:<br>

The example at [[file:examples/pin-specific-build/][examples/pin-specific-build/]] pins its own Zig build and compiles a tiny
Zig program that prints the compiler version used, then compares it with
the version pinned in the example.

Build and run it through Nix:

#+begin_src sh
  nix build ./examples/pin-specific-build#hello --no-link
  nix run ./examples/pin-specific-build#hello
#+end_src

Check the pinned Zig CLI directly:

#+begin_src sh
  nix shell ./examples/pin-specific-build#zig
  zig version
#+end_src

Or run the example's ~build.zig~ with that pinned compiler:

#+begin_src sh
  cd examples/pin-specific-build
  nix shell .#zig -c zig build run
#+end_src

The output shows the compiler version in use next to the example pin. A
plain ~zig build run~ uses whatever Zig is on your =PATH=, so a mismatch
there is expected — the Nix commands above use the pinned compiler.

** 📥 Installation

#+html:<details>
#+html:<summary><b>Add zignix as a flake input</b></summary>
#+html:<br>

#+begin_src nix
  {
    inputs.nixpkgs.url = "github:NixOS/nixpkgs?ref=nixos-unstable";
    inputs.zignix.url = "github:withre/zignix";
  }
#+end_src

You do not need to clone this repository. Nix fetches zignix from GitHub
as part of normal flake evaluation.

After that, pick the pattern for your flake shape. Frameworks like
blueprint or flake-parts hand each file a ~pkgs~ already, so the Quick
Start snippet works as-is. A plain flake needs the small helper below.

#+html:</details>

#+html:<details>
#+html:<summary><b>Plain flake without a framework</b></summary>
#+html:<br>

A plain flake needs to instantiate nixpkgs per system before calling
zignix.

#+begin_src nix
  {
    inputs.nixpkgs.url = "github:NixOS/nixpkgs?ref=nixos-unstable";
    inputs.zignix.url = "github:withre/zignix";

    outputs = { nixpkgs, zignix, ... }:
      let
        systems = [ "x86_64-linux" "aarch64-linux" "x86_64-darwin" "aarch64-darwin" ];
        forAllSystems = nixpkgs.lib.genAttrs systems;
      in {
        devShells = forAllSystems (system:
          let
            pkgs = nixpkgs.legacyPackages.${system};
            zig = zignix.lib.${system}.fromBuild {
              version = "0.17.0-dev.813+2153f8143";
              sha256 = "sha256-sNRv/EWHuejdC1JO5bxNoeZ/KLulXnxTTOxkry8tenQ=";
            };
          in {
            default = pkgs.mkShell { packages = [ zig ]; };
          });
      };
  }
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><b>Home Manager — what is available</b></summary>
#+html:<br>

To install Zig globally on your profile, add a zignix package to
~home.packages~. You can use a repo-maintained build or your own pin (all
keyed by ~${pkgs.stdenv.hostPlatform.system}~):

| What you want                   | How to reference it                                  |
|---------------------------------+------------------------------------------------------|
| Latest master (repo-maintained) | ~zignix.packages.${system}.zig-master~                 |
| Latest 0.16 (repo-maintained)   | ~zignix.packages.${system}.zig-0_16~                   |
| Latest master via overlay       | ~pkgs.zignix.master~ (after adding the overlay)        |
| Latest 0.16 via overlay         | ~pkgs.zignix."0.16"~ (after adding the overlay)        |
| A specific master build         | ~zignix.lib.${system}.fromBuild { version; sha256; }~  |
| A specific tagged release       | ~zignix.lib.${system}.fromTagged { version; sha256; }~ |
| The same compiler, renamed      | ~zignix.lib.${system}.withName "<cmd>" <pkg>~          |

#+html:</details>

#+html:<details>
#+html:<summary><b>Home Manager — repo-maintained builds</b></summary>
#+html:<br>

No version or hash to write yourself. Use the ~zig-master~ package for
the latest nightly or ~zig-0_16~ for the latest 0.16 release.

#+begin_src nix
  # home.nix
  { pkgs, zignix, ... }:
  {
    home.packages = [
      zignix.packages.${pkgs.stdenv.hostPlatform.system}.zig-master
      # or: zignix.packages.${pkgs.stdenv.hostPlatform.system}.zig-0_16
    ];
  }
#+end_src

Or via the overlay, which exposes ~pkgs.zignix.master~ and
~pkgs.zignix."0.16"~ (pass ~zignix~ to the module so the overlay is in
scope):

#+begin_src nix
  # home.nix
  { pkgs, zignix, ... }:
  {
    nixpkgs.overlays = [ zignix.overlays.default ];
    home.packages = [ pkgs.zignix.master ];
  }
#+end_src

The version you get is whatever the zignix revision in your ~flake.lock~
pins. Move to a newer build with:

#+begin_src sh
  nix flake update zignix
  home-manager switch
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><b>Home Manager — a specific version</b></summary>
#+html:<br>

Pin it yourself with ~fromBuild~ (nightly) or ~fromTagged~ (release). See
*Pin values* for finding the hash.

#+begin_src nix
  # home.nix
  { pkgs, zignix, ... }:
  let
    zig = zignix.lib.${pkgs.stdenv.hostPlatform.system};
  in
  {
    home.packages = [
      (zig.fromTagged {
        version = "0.16.0";
        sha256 = "sha256-cOSWZKdDdLSLUebz/fv0N/Y5XUJQkFBYi9SavlK6PQA=";
      })
    ];
  }
#+end_src

#+html:</details>

#+html:<details>
#+html:<summary><b>Home Manager — several versions side by side</b></summary>
#+html:<br>

Every zignix package installs a binary called ~zig~, so two of them would
collide on ~PATH~. Use ~zignix.lib.${system}.withName~ to expose each
under its own command name. You decide which one is the bare ~zig~.

#+begin_src nix
  # home.nix
  { pkgs, zignix, ... }:
  let
    zig = zignix.lib.${pkgs.stdenv.hostPlatform.system};
    pkgsFor = zignix.packages.${pkgs.stdenv.hostPlatform.system};
  in
  {
    home.packages = [
      # zig is master here; swap to pkgsFor.zig-0_16 if you prefer stable
      (zig.withName "zig"        pkgsFor.zig-master)
      (zig.withName "zig-master" pkgsFor.zig-master)
      (zig.withName "zig-0.16"   pkgsFor.zig-0_16)
    ];
  }
#+end_src

~withName~ takes any zignix package, including a ~fromBuild~ /
~fromTagged~ pin, so you can name your own versions the same way.

** 🔢 Pin values

#+html:<details>
#+html:<summary><b>Get the latest version and sha256</b></summary>
#+html:<br>

Zig publishes the latest nightly under ~.master~ in
[[https://ziglang.org/download/index.json][ziglang.org/download/index.json]]. The index stores hex ~shasum~ values,
so convert the one for your system to Nix SRI format before using it as
~sha256~.

#+begin_src sh
  # version
  curl -s https://ziglang.org/download/index.json \
    | jq -r '.master.version'
  
  # sha256 for x86_64-linux
  curl -s https://ziglang.org/download/index.json \
    | jq -r '.master."x86_64-linux".shasum' \
    | xargs nix hash convert --hash-algo sha256 --to sri
#+end_src

System key mapping:

| zignix system   | Zig index key |
|-----------------+---------------|
| ~x86_64-linux~    | ~x86_64-linux~  |
| ~aarch64-linux~   | ~aarch64-linux~ |
| ~x86_64-darwin~ | ~x86_64-macos~  |
| ~aarch64-darwin~  | ~aarch64-macos~ |

#+html:</details>

#+html:<details>
#+html:<summary><b>Let Nix tell you the expected sha256</b></summary>
#+html:<br>

If you do not want to convert the upstream shasum yourself, use a fake
hash first:

#+begin_src nix
  zignix.lib.${system}.fromBuild {
    version = "0.17.0-dev.813+2153f8143";
    sha256 = zignix.lib.${system}.fakeHash;
  }
#+end_src

Then run ~nix build~. Nix fails with the expected ~sha256~ value, which
you can paste back into the pin.

** 🧩 Other patterns

#+html:<details>
#+html:<summary><b>Tagged releases and custom URLs</b></summary>
#+html:<br>

~fromBuild~ is for Zig ~master~ builds under ~https://ziglang.org/builds~.
For tagged releases and custom mirrors:

#+begin_src nix
  zignix.lib.${system}.fromTagged {
    version = "0.16.0";
    sha256 = "sha256-cOSWZKdDdLSLUebz/fv0N/Y5XUJQkFBYi9SavlK6PQA=";
  }
#+end_src

#+begin_src nix
  zignix.lib.${system}.fromUrl {
    version = "0.17.0-dev.612+abc";
    url = "https://my-mirror/zig-x86_64-linux-0.17.0-dev.612+abc.tar.xz";
    sha256 = "...";
  }
#+end_src
